# Incident Log — {project_name}

> **類型**: 活文件 🌱（只追加不覆蓋）
> **目的**: 問題排除記錄，含三層分析
> **規則**: 每次 bug fix 後必填，序號遞增

---

## 格式（S1/S2 嚴重問題用標準版）

```
### INC-XXX: [簡短標題] (YYYY-MM-DD HH:MM)
- **現象**: 用戶看到什麼
- **嚴重性**: S1~S4
- **三層分析**:
  - **基礎層（架構）**: 系統架構面的根本原因
  - **邏輯層（程式）**: 程式邏輯哪裡出錯
  - **實作層（修復）**: 具體改了哪些檔案的哪幾行
- **解法**: 改了什麼
- **影響檔案**: 含行號
- **預防**: 未來如何避免
- **關聯版本**: vX.X.X
```

## 格式（S3/S4 小問題用簡易版）

```
### INC-XXX: [簡短標題] (YYYY-MM-DD HH:MM)
- **現象**: 用戶看到什麼
- **根因**: 為什麼發生
- **解法**: 改了什麼
- **影響檔案**: 哪些檔案
- **關聯版本**: vX.X.X
```
